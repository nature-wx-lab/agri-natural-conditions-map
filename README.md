# 気象と土地条件でみる農業適地MAP

全国1,892市町村・政令指定都市区の農業に関わる自然条件を、1つの地図で分野横断して比較するNatureWxLabの購入者向けツールです。閲覧には、対応するnote有料部分に記載された共通パスワードが必要です。

## 表示できる分野

- 水稲（共通条件と主要10品種）
- 露地野菜（葉茎菜6・果菜6・根菜／球根菜4の計16品目。3タイプを個別に切替し、総合順位は作成しません）
- 果樹（共通条件と、りんご・みかん・ぶどう・もも・なし・かき・うめ・さくらんぼ・くり・キウイフルーツ）
- 畑作（共通条件と主要10品目。露地野菜の根菜／球根菜と重複させません）
- 施設園芸（共通条件と主要10品目。外気候負荷だけの参考評価）
- 4分野総合（3つの比較方法）
- 温暖化による変化（5分野、5段階の頑健変化帯）
- 土壌分類（農耕地で面積最多の土壌大群。適性点ではない参考レイヤー）
- 気候ものさし（平均気温、最高・最低気温の平均、降水量、日照時間、最深積雪、降雪量、全天日射量の独立参考レイヤー）

「土地条件」は地形（標高・傾斜・平坦さ）と土壌を指し、交通・流通・地価などの立地条件を意味しません。この指数は収量、利益、成功確率、公的な適地認定、個別圃場の適否を示すものではありません。現在の農地位置を評価対象の選定に使いますが、作付実績、生産量、収量、ブランドは得点に使っていません。46品目のうち施設7品目の作物別夜温最低限界・昼温最高限界は農林水産省マニュアル表2に基づきますが、それ以外の品目別閾値と全分野の変換幅・施設外気候変換は、一次資料で検証済みの栽培基準ではない非査読・著者設定の探索パラメータです。購入者ExcelとMAP自治体詳細で全閾値・出典区分・重複条件・感度範囲を開示します。品目別56ビューの主得点は、評価できる対象農地全体を面積加重して集計します。品目ごとの必須気温・作期・冬季条件は、土壌など別の良い条件で相殺せず、最も厳しい制約を主得点へ反映します。局所的に条件の良い農地を示す上位20％の値は参考情報として分離しています。主な制約要因は自治体内で面積加重平均した3成分の最小値で、メッシュごとの最頻要因ではありません。感度順位は変換幅だけを0.8倍・1.0倍・1.2倍した3条件で、モデル全体の不確かさではありません。水稲は全国作付割合、露地野菜・畑作は全国収穫量、施設園芸は施設内栽培延べ面積を、各部門内の品目選定だけに使用します。異なる単位は合算せず、得点の校正にも使いません。施設園芸は外気候負荷だけの参考評価で、施設内の気候制御、燃料、設備、風・雪・雹への構造安全性を評価しません。果樹品目別表示では、2020年農林業センサスの販売目的栽培面積を「実際の産地分布の目安」として独立した静止状態の紫色市町村境界線で重ねられます。これは適地得点へ使わず、収穫量や収量を表すものでもありません。土壌分類は、ランキングに使う土壌軸とは別の参考表示です。

地図には日本周辺の国・地域の境界を表示します。「地図表示」から地点名、国土地理院の詳細地図、地形を任意に重ねられ、詳細地図と地形はそれぞれ濃さを0〜100％で調整できます。地点名・詳細地図・地形は初期状態ではOFF、地形の初期スタイルは白黒で、カラーにも切り替えられます。これらの表示状態は共有URLに保存されます。詳細地図または地形をONにした場合だけ `cyberjapandata.gsi.go.jp` へタイル画像を取得し、通常のWeb通信として閲覧者のIPアドレス等が同サーバーへ送られます。Referrerは送信しません。

分野は左パネルのプルダウンで切り替えます。地図内左上の `＋`、`−`、`全` で拡大・縮小・全国表示へ戻せます。デスクトップの全国表示は本土を中心に見やすく配置し、スマートフォンでは全体が収まる表示を維持します。ヘッダーの「解説記事」は、農業適地ランキングシリーズのnoteマガジンへ接続します。

「気候ものさし（参考）」では、気候ものさしナビで検証済みの全国1km気候面から、8要素・2つの30年平均期間を表示します。平均気温、最高・最低気温の平均、降水量、日照時間、最深積雪、降雪量は1〜12月と年平均／年合計を切り替えられます。全天日射量は元データに年値がないため1〜12月だけを表示し、ブラウザ内で年値を合成しません。濃さは0〜100％で調整できます。表示する1枚だけを必要時に読み込むため、206枚・合計6,878,124 bytesの気候画像を起動時に一括取得しません。

地図へポインターを重ねると、選択中の要素・月・期間の1km地点値をツールチップへ表示します。クリックまたはタップすると1kmメッシュを固定し、右パネルへ同じ月・年の全8要素について1991〜2020年、1996〜2025年、両者の更新差を表示します。更新差は25年間が重なる2つの30年平均の差で、30年間の変化量そのものではありません。固定地点は `climateMesh` として共有URLへ保存します。

地点値は、同一originの気候ものさしナビが公開済みの検証済みchunkから必要分だけ取得します。農業MAP側へ257,482,808 bytes・1,408 chunkを複製せず、ホバー時は選択要素の1件、地点固定時は最大8件を遅延取得します。農業MAPのmanifestは各chunkのbytes・SHA-256・件数・メッシュ範囲を固定し、ブラウザはSHA-256一致を確認してから値を読みます。不一致時は値を表示せず明示的に失敗します。

気候面と地点値はNatureWxLabが気象庁観測値から独自算出・独自内挿した参考表示です。自治体の平均値や順位ではなく、農業適地の得点・順位・閾値・暗号化payloadにも使いません。表示値は選択した1kmメッシュの値で、自治体全域の代表値ではありません。

デスクトップの左側にある表示条件パネルと、右側にある自治体詳細・全国ランキングのパネルは、地図カードと同じ下端までの高さに固定し、続きは各パネル内のスクロールバーで閲覧します。ページ全体の高さを長い操作項目や詳細情報へ合わせないため、地図下部に空のカード領域を作りません。狭い画面では固定高を解除し、通常のページスクロールで表示します。

デスクトップの地図見出しは、選択中の分野・表示指標・自治体数を横一列へまとめます。見出しを縦2段にせず、削減した高さを地図本体へ割り当てます。

地図下の読み方6項目は、デスクトップでは短い要点を横1段にまとめます。狭い画面では2列または1列へ切り替え、可読性を維持します。

拡大時は自治体の塗り面を同色の細い下線で補い、その上から行政界を描き直します。簡略化した隣接ポリゴン間に背景色の細い隙間が見えるのを防ぎつつ、境界線は維持します。

## データ

- 気象庁「過去の気象データ・ダウンロード」
- 国土交通省「国土数値情報 行政区域データ」2024年版
- 国土交通省「国土数値情報 標高・傾斜度3次メッシュ」
- 農研機構「日本土壌インベントリー 全国農耕地土壌図」（CC BY 4.0）
- 農林水産省「2020年農林業センサス」市区町村別・販売目的の果樹類の品目別栽培面積（e-Stat）
- 農林水産省「米の品種別作付動向」「作物統計」「野菜生産出荷統計」「園芸用施設の設置等の状況」
- Natural Earth（日本周辺の国・地域境界、Public Domain）
- 国土地理院「地理院タイル」（標準地図、陰影起伏図、色別標高図）
- 気象庁の観測値をNatureWxLabが独自算出・独自内挿した全国1km気候面（1991〜2020年、1996〜2025年）

公開データをNatureWxLabが独自加工しています。土壌図の出典表記は「農研機構日本土壌インベントリー（NARO, Japanese Soil Inventory）」です。各機関が本ツールを作成、検証、推奨したものではありません。行政界は地図表示用に簡略化しています。土壌分類は全国農耕地土壌図と2024年行政界の面積交差から市町村ごとの最多土壌大群を求めたもので、圃場ごとの土壌や土壌の良し悪しを示しません。

気候参考レイヤーの生成正本は `scripts/build_agri_climate_reference.py`、公開manifestは `data/climate/manifest.json` です。入力元 `public/climate-outlook-navi/data/climate/` は読み取り専用とし、検証済みdataset ID `climate-baseline-all-9c5b7a1f6c29f0f9`、generator `2.0.2` の絶対値WebPだけを農業MAPへ複製します。地点値chunkは農業MAPへ複製せず、同一originの気候ものさしナビ配信をSHA-256検証付きで参照します。季節予報、差分画像、元CSVは農業MAPへ含めません。

- 日本土壌インベントリー: https://soil-inventory.rad.naro.go.jp/
- デジタル土壌図の利用条件: https://soil-inventory.rad.naro.go.jp/pdf/soil_map_application.pdf

## アクセス方式

地図データは圧縮後にAES-256-GCMで暗号化して配信します。パスワードと復号処理は閲覧者のブラウザ内だけで扱い、外部へ送信しません。「このブラウザで次回から自動的に開く」を選ぶと、抽出不可のWeb Crypto `CryptoKey` を同じブラウザへ保存します。生鍵バイト形式で保存済みの記録は初回解錠時に `CryptoKey` へ移行して置き換えます。共通パスワードそのものは、このリポジトリへ保存していません。
